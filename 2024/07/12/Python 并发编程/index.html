学习Python并发编程的记录

写在前面

最近实习工作中经常碰到实验平台起任务时，显卡不够用，要排队很长时间。对于负责的测评任务，只有在前半程生成模型回复时才需要使用显卡资源，而后半程的打分只需要CPU处理，有时任务数据量大，打分时间过长会导致提示任务Hang住了，浪费线上显卡资源。因此，快速的输出评分结果能够尽快释放资源，提高实验效率。借这么个机会，了解学习 Python 并发编程。

Python 程序提速方法

  1. 单线程串行：不加改造的Python程序
  2. 多线程并发：threading模块，使多个线程在同一进程并行运行，利用CPU和IO可以同时执行的原理，让CPU不会等待IO完成
  3. 多CPU并行：multiprocessing模块，在多个进程之间分配任务，有效利用多核CPU优势，真正的并行执行任务
  4. 多机器并行：Hadoop/Hive/Spark等框架，将任务分配到多个机器上并行执行

Python 对并发编程的支持

除了上面提到的threading和multitiprocessing模块，Python 还对并发编程提供了更高级的支持：

  • 支持 asyncio 模块，在单线程利用CPU和IO同时执行的原理，实现函数粒度的异步编程。
  • 使用Lock对资源加锁，防止多线程同时访问共享资源，避免数据不一致问题。
  • 使用Queue实现不同线程/进程间通信，实现生产者-消费者模式。
  • 使用线程池/进程池，简化线程/进程的任务提交、等待结束、获取结果。
  • 使用subprocess模块启动外部程序的模块，并进行输入输出交互。

多线程 Thread | 多进程 Process | 多协程 Coroutine？

在开始之前，先要了解任务的类型，才能选择合适的并发编程形式。

任务类型

  • CPU密集型：计算密集型，指I/O在很短时间就可以完成，而CPU需要大量的计算和处理，如：压缩解压、加密解密、正则表达式搜索等。

  • IO密集型：指系统运作大部分状况是CPU在等IO的读/写操作，CPU占用率较低，如：文件处理程序、网络爬虫、读写数据库等。

多线程、多进程、多协程的对比

  • 首先，一个进程中可以启动多个线程，而一个线程中又可以启动多个协程。
  • 多线程（最常用）：适用于IO密集型计算，同时运行的任务数目要求不多
    • 优点：相比进程，更轻量级，占用资源少
    • 缺点：相比进程，多线程只能并发执行，不能利用多CPU（GIL）；相比协程，启动数目有限制，占用内存资源，有线程切换开销
  • 多进程：适用于CPU密集型计算
    • 优点：可以利用多核CPU并行运算
    • 缺点：占用资源最多，可启动数目比线程少
  • 多协程：适用于IO密集型计算，需要超多任务运行，有现成库支持的场景
    • 优点：内存开销最小，启动协程数量最多
    • 缺点：支持的库有限制，代码实现复杂

Python速度慢的"罪魁祸首"

Python相比于C/C++等静态语言，在某些特定任务上可能速度相差百倍，造成速度慢的原因如下：

  • Python是解释型语言，由解释器边解释边执行，而不像编译型语言那样在执行之前将代码编译成机器语言。
  • Python是动态型语言，变量的类型在运行时才确定，这使得每次操作都要检查变量的类型，并且在执行过程中引入额外的开销。
  • GIL(Global Interpreter Lock) 是用于同步线程的一种机制，使得同一时刻只有一个线程执行Python字节码，无法利用多核CPU并发执行能力。

怎样规避GIL带来的限制？

  1. 多线程机制仍然有用，用于IO密集型计算

    1. 因为在IO期间，线程会释放GIL，实现CPU和IO的并行，因此多线程用于IO密集型计算依然可以大幅度提升速度
    2. 但对于CPU密集型计算，只会更加拖慢速度，因为在不同线程之前切换也需要开销

  2. 使用multitiprocessing模块的多进程机制，利用多核CPU的并行计算，实现真正的并行计算。

Queue 模块

在多线程和多进程编程中，常常需要在线程或进程之间传递数据。如果直接通过共享变量进行数据传递，会引发竞争条件、数据不一致等问题。为了解决这个问题，Python提供了queue模块，用于安全地在线程和进程之间进行数据传递。

Queue 的主要功能

  • 线程/进程安全：Queue实现了锁机制，能够保证在多个线程/进程访问时，数据的一致性和安全性。
  • 生产者-消费者模式：在这种模式中，生产者线程/进程不断生产数据并放入队列，而消费者线程/进程不断从队列中取数据进行处理。

Python 四种队列类型

  • Queue：先进先出（FIFO），即先放入队列的数据先被取出。
  • LifoQueue：后进先出（LIFO），即最后放入队列的数据先被取出。
  • PriorityQueue：优先级队列，根据元素的优先级进行排序，优先级高的元素先被取出。
  • SimpleQueue：简单队列，不保证队列元素的顺序，适用于不需要顺序的场合。
  • deque：双端队列，可以从两端进行添加和删除操作，适用于需要频繁从两端进行数据操作的场合。

###　队列常用的方法

  • put(item)：将一个元素放入队列中。
  • get([block[, timeout]])：从队列中取出一个元素，block是否阻塞等待，timeout等待时间。
  • get_nowait()：相当于get(False)，非阻塞方法，则立即返回Empty异常。
  • empty()：判断队列是否为空。
  • full()：判断队列是否已满。
  • join：阻塞直到队列中的所有任务都被处理完毕。

##　线程安全
在多线程编程中，由于多个线程可以同时访问共享资源，因此可能会出现竞争条件，导致数据不一致或程序崩溃。为了解决这个问题，Python提供了多种线程同步机制，包括锁、信号量、条件变量等。

锁（Lock）

锁是最常用的线程同步机制，它允许多个线程同时访问共享资源，但一次只能有一个线程持有锁。其他线程在尝试获取锁时会被阻塞，直到锁被释放。
